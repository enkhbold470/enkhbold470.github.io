# SvelteKit Migration Design

**Date:** 2026-05-26
**Status:** Approved, ready for implementation plan

## Goal

Replace the existing Next.js 16 + React 19 implementation of `enk.icu` with a SvelteKit application using Bun for all package management, scripts, and the production runtime. Functionality, content, and visual output stay identical.

## Approach

Replace the Next.js codebase in-place on a new `svelte-migration` branch. `main` is untouched until merge. No side-by-side directory.

## Stack

- **Framework:** SvelteKit (latest), scaffolded with `bunx sv create` using the minimal TS template.
- **Adapter:** `@sveltejs/adapter-node`. Production command: `bun ./build/index.js`.
- **Styling:** Tailwind v4 via `@tailwindcss/vite`. No `tailwind.config.js`, no `postcss.config`. Project-wide CSS lives in `src/app.css`.
- **Package manager / runtime:** `bun` exclusively — `bun install`, `bun run dev`, `bun run build`, `bunx sv ...`. No npm, pnpm, or yarn invocations.

## Library Swaps

| Next.js dependency | SvelteKit replacement |
|---|---|
| `next/font/google` (Noto Serif) | `@fontsource/noto-serif` |
| `next-themes` + custom `ThemeProvider` | Hand-rolled `ThemeToggle.svelte` — class on `<html>`, persisted in `localStorage`, pre-hydration inline script in `app.html` to prevent flash of incorrect theme |
| `lucide-react` | `lucide-svelte` |
| `@vercel/analytics/next` | `@vercel/analytics/sveltekit` |
| `next/image` (`unoptimized` on blog list) | plain `<img>` |
| `next/link` | plain `<a>` (SvelteKit intercepts internal links by default) |
| `Metadata` export | `<svelte:head>` per page |
| `rss-parser` | unchanged — works under Bun's Node compat |

## Route Map

```
src/routes/
  +layout.svelte              # font import, theme init, <ThemeToggle/>, <StructuredData/>, Analytics
  +page.svelte                # home page
  +error.svelte               # 404 / error UI
  blog/
    +page.server.ts           # load: fetchMediumFeed()
    +page.svelte              # post list
    [slug]/
      +page.server.ts         # load: find post by slug, transform YouTube links server-side
      +page.svelte            # post body via {@html ...}
  sitemap.xml/+server.ts      # XML response, includes Medium slugs
  robots.txt/+server.ts       # text/plain response
```

## File Migrations

### Preserved verbatim (relocated only)

- `config/siteConfig.ts` → `src/lib/siteConfig.ts` (no logic change; imports become `$lib/siteConfig`)
- `utils/mediumFeed.ts` → `src/lib/mediumFeed.ts` (no logic change)
- `app/globals.css` → `src/app.css` (replace `@tailwind base; @tailwind components; @tailwind utilities;` with the single Tailwind v4 line `@import "tailwindcss";` — every other rule is preserved byte-for-byte)
- `public/` → `static/`

### Rewritten (semantics preserved)

- **`utils/renderBio.tsx`** → inline in home page. A `{#each}` walk over `siteConfig.bio.links` reuses the existing `[linkText]` parsing logic.
- **`components/StructuredData.tsx`** → `src/lib/StructuredData.svelte`. Emits `<script type="application/ld+json">{@html JSON.stringify(schema)}</script>` inside `<svelte:head>`. The two schema objects (`Person`, `ProfilePage`) move over unchanged.
- **`components/ThemeToggle.tsx`** → `src/lib/ThemeToggle.svelte`. On mount reads `localStorage.theme` (falls back to `prefers-color-scheme`) and applies `.dark` to `<html>`. Click toggles and writes back to `localStorage`. A pre-hydration inline `<script>` in `src/app.html` sets the class before paint to avoid theme flash.
- **`components/ThemeProvider.tsx`** → deleted. Replaced by the inline script + `ThemeToggle` lifecycle.

## Behavior Changes (intentional)

1. **Blog data fetching.** Next's `dynamic = 'force-dynamic'` + `revalidate = 60` becomes a `+page.server.ts` `load` function that calls `setHeaders({'cache-control': 'public, max-age=60'})`. Same observable UX, native SvelteKit primitive.
2. **YouTube link rewriting** moves from render-time (`app/blog/[slug]/page.tsx`) into the server `load`. The HTML is already produced server-side, so the transform belongs there.

## Explicitly NOT Ported

- The commented-out `<script async src="https://unpkg.com/...">` block in `app/layout.tsx`. It is commented and unused.

## Risks

- **`rss-parser` on Bun.** Uses Node streams via Bun's Node-compat layer. Expected to work. If it misbehaves, fallback is native `fetch` + regex-based parsing of the Medium RSS XML. Low probability.
- **Medium feed unavailable.** Already handled in `mediumFeed.ts` (returns `[]` on error). Blog page already renders an empty state.

## Verification

After implementation, the smoke test is:

```
bun run build
bun ./build/index.js
```

Then visit `/`, `/blog`, `/blog/<some-slug>`, `/sitemap.xml`, `/robots.txt`, and a nonexistent route. Confirm:
- Home renders bio with inline links and recent updates.
- Blog list shows Medium posts (or empty-state if the feed is down).
- A blog slug page renders post HTML with YouTube embeds working.
- `/sitemap.xml` contains both static routes and Medium slugs.
- `/robots.txt` returns the expected text.
- The 404 route renders the error UI.
- The theme toggle works, persists across reload, and produces no flash on initial paint.
